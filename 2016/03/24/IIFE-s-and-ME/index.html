IIFE's and ME!

I think it’s finally happening. For the past couple of months I have been diving in hard to learn Javascript. Web development is always something I’ve been interested in. I can recall my first website, a fan page of Banjo-Kazooie that was hosted on Fortunecity or Geopages. One of those big city-esque site hosts from the 90s. But I’ve always avoided the real programming aspect of web development.

I started programming very casually a year ago in Ruby. It just seemed like a nice easy to use language that worked well for what I was doing. I spent months creating a bot for Twitch. It was a fairly simple premise, people say commands and it runs some logic. There were a few advanced things in there, at least to me such as running automated games that viewers could play. Then I went back to web development, something I hadn’t touched seriously before. I created Talutha.tv.

It used a simple Ruby backend framework called Sinatra which allowed me to run some logic to get information from a PostgreSQL database I had setup through Digital Ocean where my bot was also hosted(my website was hosted through Heroku). All in all, I created what I ultimately set out to do, I created a website that viewers could see their stats on in real time while interacting with my bot through Twitch. It was awesome. But there were still things I couldn’t do. Things that involved Javascript. Being an amateur programmer at this point, I felt I didn’t want to learn another programming language and ultimately let the whole project grow stagnant.

Fast forward to today, I decided a couple of months ago I really wanted to get into this whole development stuff. I had a LOT of fun creating my bot and website, but I wanted to do more. I wanted to learn Javascript so I could truly do some cool stuff on the internet. And here we are. Its been tough, Javascript is definitely a much more difficult language to learn than Ruby ever was, but I think I’ve finally started to have break throughs.

It started earlier today when I wondered what the difference was between a function expression and a function decleration. I mean, I understand the basic concept, an expression uses a var keyword before the function and a decleration just puts the function with no var keyword.

Why would I want to use one over the other? So I went onto Google and searched around for a bit before coming across this article by Angus Croll. I read it, and I understood it. My first major breakthrough in Javascript I felt where I actually read through an entire article and felt like I came out the other end smarter than before rather than more confused. I already understood how hoisting works differently with expressions and declerations which helped immensly. I’m getting somewhere, and I’m excited.

That isn’t the topic of today’s blog entry though, maybe I’ll go back and talk about when to use what in reference to functions but today we’re going to talk about my second breakthrough(in the same day even!): IIFE’s, or Immediately-Invoked Function Expressions. Please keep in mind though that I write these blogs to further my own understanding of everything so my writing might be a little odd at times but if you have any questions or comments, feel free to email me and I will do my best to help you out.

IIFEs are really cool and I can’t wait to figure out ways to use them within my own code. Now that I’ve made these couple of breakthroughs in my Javascript knowledge, I’m more motivated than ever. If you’re learning folks, don’t give up! Who knows when you’ll hit that first breakpoint. Good luck!
