Apply(), Call(), and Bind() Walk Into This

  • -Javascript -Learning

Over the course of yesterday and today I have been learning about Closures and Callbacks. Lucky for me neither concept was difficult to understand and after a bit of practice I think I can start to see ways to use both of these tools. But these two things led directly into my next lesson which did give me a bit of trouble. I’m hoping by writing this down and going over it again I will solidify these concepts in my mind and hopefully help a fellow lost programmer along the way. These concepts, or rather methods, are Apply(), Call(), and Bind().

These three methods are actually pretty easy to understand and you can do some pretty neat things with them. We’ll start out with Bind.

We just created a person object that holds firstName, lastName and a method, getFullName. getFullName is taking advantage of the this keyword which should reference the containing function, thus this.firstName should grab Marvin. Next we setup a new function called logName that takes the full name and outputs it to the console. But something doesn’t quite add up here. In logName we are trying to invoke this.getFullName(), but will this work? The answer is no because this in logName is pointing to the global execution context and in no way points to the person object.

So can we make this work? Of course! Or else I wouldn’t be writing this blog post! This is where those three methods from earlier come in. Here is how we can apply bind() to the situation.

You can see that we created a new variable, logPersonName, and set it equal to logName.bind(person). Then we invoked logPersonName(); instead of logName. But where did bind() come from? We all know that all function objects in Javascript, or any language really, come with methods out of the box. That is how we get things like .length and .toLowerCase. bind(), apply(), and call(); are all methods that are automatically given to all functions.

In the above example for bind() we are basically changing where this is pointing. Instead of pointing to the containing function, which in the case of logName would be the global execution context, it is now pointing to person. bind() has done this by copying the logName function entirely creating a new execution context when it is invoked.

Lets do the same thing with call, but we’re going to change it up a little bit.

call() works very similar to bind(). The biggest difference is that call() does not copy the function, it runs it in place with thispointing to the correct place. Another thing you might notice here is where we are telling it to point this to. It is the first argument. We shifted the arguments of logName over by one and added the reference to the front.

apply() is the exact same thing as call(). It is so similar in fact that I’m going to be lazy and only copy/paste the one line that matters.

The only difference with apply() is that the arguments must be in an array. Which method you use of the two really depends on the type of data you are trying to pass along, so go with whichever you feel more comfortable with!

These concepts alone I don’t feel are too confusing. The concepts they lead into though can cause a bit of head scratching. Lets take a look at Function Borrowing.

Whoa, so what all is going on here? In our program, we want multiple person variables but we don’t want to reuse code over and over again. After all, isn’t the point of programming to make our lives easier and to remove all the busy work? So we want to borrow as much code as we can, in this case we are borrowing functions!

Remember from earlier what person.getFullName contained.

It contains a function, a function referencing this. In this case, when we use apply(), we are moving the this reference from person to person2 where it will then get the first and last names, thus outputting Mary Sue to the console. This can be very confusing, and in fact it is still a little confusing to me as I sit here and type it which is why I am doing so. Just try to remember, person.getFullName is only returning a function with references to this. We simply are telling it to look to person2 instead of person with .apply(person2). Nice!

The final topic of the night is called Function Currying. Here we go.

It looks like there is a lot of stuff happening here, but really there is only one or two things happening that would seem confusing. We create a function, multiply and pass it two arguments which it then returns the product of. Next we create a new variable, multiplyByTwo which is where we apply Function Currying. We use bind() in this case with two arguments. But remember that bind() is taking the first argument like we saw earlier with call() and apply(). Are you starting to see how similar all of these methods are now?

In this specific case, we want to take advantage of the fact that bind() copies the function, because our multiplyByTwo function is something we want to use again and again. Now because bind() is taking the first argument, the only argument left is 2, which goes into the a variable of multiply. Now anytime we call multiplyByTwo we only have to pass one argument because the first argument will always be 2. In our specific example, we pass 5 which gives us 10 because 2 * 5 is 10.

If we were to not use Function Currying, it would be exactly the same as us using multiply(2, 10);, but instead we just use multiplyByTwo(5). That is Function Currying.

Hopefully this wasn’t terribly confusing for you. It certainly did help me out a lot to type this out. None of these methods or ideas should scare you away. While they look daunting at first, they are very powerful tools to help us write beautiful looking and powerful Javascript. So code away!
